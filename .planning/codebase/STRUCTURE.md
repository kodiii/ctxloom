## Directory Purposes

**`src/`:**
- Purpose: All TypeScript source code; compiled to `dist/` by tsup
- Contains: Entry point, server, and all subsystem modules
- Key files: `src/index.ts` (CLI), `src/server.ts` (MCP server and all tool handlers)

**`src/ast/`:**
- Purpose: AST-based code analysis using web-tree-sitter WASM
- Contains: Parser for TypeScript/JavaScript nodes, Skeletonizer for token-efficient file summaries
- Key files: `src/ast/ASTParser.ts`, `src/ast/Skeletonizer.ts`

**`src/db/`:**
- Purpose: Vector embedding persistence and approximate nearest neighbor search
- Contains: Single `VectorStore` class wrapping LanceDB
- Key files: `src/db/VectorStore.ts`

**`src/graph/`:**
- Purpose: Static import dependency analysis across the whole project
- Contains: Bidirectional in-memory graph with JSON snapshot persistence and symbol index
- Key files: `src/graph/DependencyGraph.ts`

**`src/indexer/`:**
- Purpose: Local ML embedding generation and batch directory indexing
- Contains: HuggingFace Transformers pipeline wrapper, file collector, batch indexer
- Key files: `src/indexer/embedder.ts`

**`src/security/`:**
- Purpose: Centralized file system access control
- Contains: `PathValidator` — validates all file paths stay within project root
- Key files: `src/security/PathValidator.ts`

**`src/setup/`:**
- Purpose: First-time onboarding and MCP client configuration
- Contains: Client registry with 12+ MCP tool definitions (Claude Desktop, Claude Code, Cursor, Windsurf, VS Code, Continue.dev, Aider, Codex CLI, etc.), interactive setup wizard, postinstall hook
- Key files: `src/setup/clients.ts`, `src/setup/setup-wizard.ts`, `src/setup/postinstall.ts`

**`src/tools/`:**
- Purpose: Reusable MCP tool handler logic separate from server routing
- Contains: Call graph traversal (BFS with configurable depth), project rule file aggregation
- Key files: `src/tools/findCallers.ts`, `src/tools/ruleManager.ts`

**`src/watcher/`:**
- Purpose: Incremental re-indexing triggered by file system events
- Contains: `FileWatcher` wrapping chokidar with debounce
- Key files: `src/watcher/FileWatcher.ts`

**`src/workers/`:**
- Purpose: Off-main-thread processing infrastructure
- Contains: Worker thread script for embedding + LanceDB upsert
- Key files: `src/workers/indexerWorker.ts`

**`tests/`:**
- Purpose: Vitest test suite; one test file per source module
- Contains: Unit and integration tests; co-located fixture files in `tests/fixtures/`
- Key files: `tests/fixtures/sample.ts` (parseable TypeScript for AST tests), `tests/fixtures/config.ts`

**`.ctxloom/` (runtime-generated, not in repo):**
- Purpose: Project-local data store created during indexing
- Contains: `vectors.lancedb` (LanceDB vector database), `graph-snapshot.json` (serialized DependencyGraph)
- Generated: Yes — created on first `ctxloom index` or server startup
- Committed: No — should be in `.gitignore`

**`dist/` (build output, not in repo):**
- Purpose: Compiled JavaScript output served as the npm package
- Generated: Yes — by `npm run build` (tsup)
- Committed: No — generated artifact; WASM files are copied here by tsup `onSuccess` hook

## Key File Locations

**Entry Points:**
- `src/index.ts`: Primary CLI entry point; dispatches to server, indexer, or setup wizard
- `src/server.ts`: MCP server definition; all six tool handlers live here
- `src/workers/indexerWorker.ts`: Worker thread entry point for off-thread embedding
- `src/setup/postinstall.ts`: npm postinstall hook entry point

**Configuration:**
- `package.json`: Package metadata, `bin` (maps `ctxloom` to `dist/index.js`), `scripts`, dependencies, `engines.node >= 20`
- `tsconfig.json`: TypeScript strict mode, `NodeNext` module resolution, outputs to `dist/`, sources from `src/`
- `tsup.config.ts`: Build entry points (`src/index.ts`, `src/workers/indexerWorker.ts`, `src/setup/postinstall.ts`), ESM format, WASM copy in `onSuccess`
- `vitest.config.ts`: Test include pattern `tests/**/*.test.ts`, 30s timeout, node environment

**Core Logic:**
- `src/server.ts`: All MCP tool routing and handler implementations
- `src/graph/DependencyGraph.ts`: In-memory graph data structure with persistence
- `src/db/VectorStore.ts`: LanceDB wrapper
- `src/indexer/embedder.ts`: Embedding generation and file collection

**Security:**
- `src/security/PathValidator.ts`: Must be used for ALL file reads exposed via MCP tools

**Testing:**
- `tests/fixtures/sample.ts`: Fixture TypeScript file with functions/classes for ASTParser tests
- `tests/fixtures/config.ts`: Fixture config for tests

## Naming Conventions

**Files:**
- PascalCase for class modules: `ASTParser.ts`, `VectorStore.ts`, `DependencyGraph.ts`, `FileWatcher.ts`, `PathValidator.ts`, `Skeletonizer.ts`, `RuleManager.ts`
- camelCase for functional modules: `embedder.ts`, `findCallers.ts`, `ruleManager.ts` (note: class file `RuleManager.ts` is PascalCase; its function-oriented counterpart would be camelCase)
- kebab-case for multi-word non-class files: `setup-wizard.ts`, `indexerWorker.ts`
- Test files mirror source name with `.test.ts` suffix: `ASTParser.test.ts`, `VectorStore.test.ts`

**Directories:**
- Lowercase, single-word or camelCase: `ast/`, `db/`, `graph/`, `indexer/`, `security/`, `setup/`, `tools/`, `watcher/`, `workers/`
- Organized by domain/subsystem, not by type (no `models/`, `controllers/`, `services/` pattern)

## Where to Add New Code

**New MCP Tool:**
1. Add Zod schema near the top of `src/server.ts`
2. Add tool descriptor to `ListToolsRequestSchema` handler in `src/server.ts`
3. Add `if (name === 'ctx_new_tool')` branch in `CallToolRequestSchema` handler in `src/server.ts`
4. If logic is complex, extract handler to a new file in `src/tools/`
5. Add test: `tests/NewTool.test.ts`

**New Subsystem/Module:**
- Implementation: `src/{domain}/ModuleName.ts` (PascalCase for class-based, camelCase for functional)
- Tests: `tests/ModuleName.test.ts`
- Register lazy singleton in `src/server.ts` if server-lifetime state is needed

**New File Type Support (indexing/watching):**
- Add extension to `SUPPORTED_EXTENSIONS` set in `src/indexer/embedder.ts`
- Add extension to `isSourceFile()` method in `src/watcher/FileWatcher.ts`

**New MCP Client Detection:**
- Add client entry to the client registry array in `src/setup/clients.ts`

**Utilities / Shared Helpers:**
- If pure utility (no subsystem dependency): add to the relevant domain module or create `src/utils/` (does not currently exist)
- Security-related path handling: extend or use `src/security/PathValidator.ts`

## Special Directories

**`.ctxloom/`:**
- Purpose: Runtime data directory created by ContextMesh in the indexed project's root
- Generated: Yes (created during `ctxloom index` or first server start)
- Committed: No (should be gitignored in consumer projects)
- Contents: `vectors.lancedb/` (LanceDB columnar store), `graph-snapshot.json` (serialized graph)

**`dist/wasm/`:**
- Purpose: WASM binaries required by `ASTParser` at runtime
- Generated: Yes (copied by tsup `onSuccess` hook from `node_modules/`)
- Committed: No (build artifact)
- Contents: `tree-sitter.wasm`, `tree-sitter-typescript.wasm`, `tree-sitter-tsx.wasm`

**`tests/fixtures/`:**
- Purpose: Static test fixture files consumed by test suites
- Generated: No (hand-authored)
- Committed: Yes

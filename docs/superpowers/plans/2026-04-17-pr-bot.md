# ctxloom — PR-Native GitHub App (Reviewer Cheatsheet Bot)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `@ctxloom-bot` — a GitHub App that on every PR `opened|synchronize|reopened` event (1) ensures a ctxloom graph is current for the PR's base + head, (2) runs `detect_changes` + `risk_overlay` + `get_affected_flows` + `git_coupling` against the diff, (3) posts a single idempotent top-level summary comment with risk score, affected flows, and suggested reviewers, (4) posts inline review comments on the highest-risk hunk per file, and (5) optionally posts a Check Run with pass/fail on `risk < threshold`. Slash commands (`/ctxloom explain|ignore|refresh`) let reviewers drive it from the PR.

**Architecture:** New npm workspace `apps/pr-bot` built on Probot. Core ctxloom analysis is first refactored into pure library functions so the bot consumes them without MCP. Per-installation on-disk cache of graph snapshots keyed by `{installationId, repoId, baseSha}`. All graph work happens in the bot's worker — only assembled markdown is sent to GitHub. Per-repo config via `.ctxloom.yml`.

**Tech Stack:** TypeScript/ESM, Probot v13, Octokit, Fastify (already transitively present), existing ctxloom core as workspace dep, vitest, Dockerfile for deploy. Depends on the sibling git-coupling plan for the "historical surprises" line in comments — plan still ships without it, just with thinner data.

- [ ] **Step 9.2: Implement**

```typescript
export async function onPullRequest(context): Promise<void> {
  const config = await loadRepoConfig(context.octokit, context.repo());
  const { rootDir, graph, overlay } = await ensureGraph({ ... });
  const payload = await buildReview({ rootDir, graph, overlay, pr: context.payload.pull_request, config });
  await upsertSummaryComment(context.octokit, context.repo(), payload.pr.number, renderSummary(payload));
  if (config.inline_comments) await submitInlineReview(context.octokit, payload);
  if (config.check_run) await publishCheck(context.octokit, payload);
}
```

## Dependencies & sequencing

- Task 2 (library extract) is the gate for everything in `apps/pr-bot` — ship it first and get its parity tests green.
- Tasks 3–5 can be worked on once Task 2 is green.
- Tasks 6–8 are independently parallelizable after Task 5.
- Task 9 requires 3–8.

- Sibling plan **git-coupling-churn** is a soft dependency — the bot ships without it (richer blocks simply render as "—"), but the "historical surprises" in the summary and inline comments need it to be compelling.

## Out of scope (deliberately)

- GitLab / Bitbucket / Gitea adapters (v2).
- LLM-authored review prose (v1 is deterministic markdown; LLM prose is a v2 opt-in toggle).
- Auto-approve / auto-merge — bot only advises.
- IDE sidecar (separate product surface).
